## 🐛 **What Bug Does V4.2 Fix?**

### **The Problem:**
```
User uploads: igračke.xls (Excel file with categories)

V4.1: LLM hallucinates answer (fabricates "Za dječake" category)

```

**Root cause:** V4.1 didn't recognize ".xls" as document query → Router chose web_search → Mixed web + binary garbage → Hallucination

**Fix:** V4.2 detects file extensions → Forces RAG → Parses Excel → Accurate answer

## 🔧 **Technical Changes**

### **V4.1 → V4.2 Code Changes:**
1. **`_isDocumentQuery()` enhancement** (~20 lines)
   - Added file extension detection (`.xls`, `.xlsx`, `.doc`, `.docx`, etc.)
   - Added new patterns: "pronađi...iz", "ima li...u", "koliko...u"

2. **Router bypass logic** (~25 lines)
   - Check for file extensions before router
   - Force RAG plan if file extension found
   - Skip LLM router entirely (0ms vs 2.9s)

3. **RAG execution enhancement** (~10 lines)
   - Added `plan.forceRag` flag
   - Priority check in `shouldUseRag` logic

**Total:** ~60 lines of code changes
